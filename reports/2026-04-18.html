Craft Agents is an open-source, document-centric desktop GUI for working with AI coding and task agents. Built by the team behind craft.do, it layers an intuitive multitasking interface on top of the Claude Agent SDK and Pi SDK, letting users run parallel agent sessions, connect to arbitrary APIs and services, and share sessions — all without the 'fluff' of traditional CLI-first agent tooling.

The project is notable for embracing 'agent-native' software principles: every part of the UI is customizable through prompts, and the team itself reportedly uses Craft Agents to build Craft Agents, effectively dogfooding a no-code-editor workflow. Released under Apache 2.0, it is one of the first polished, open-source GUIs aimed at operators and knowledge workers who want a document-shaped interface to agentic work rather than a terminal.

It is trending because it offers a legitimately new shape for consumer-facing agent tooling — one that prioritizes fluid UX, remixability, and integration with existing SDKs — making it appealing to developers exploring post-CLI agent experiences and to teams looking for an opinionated starting point they can fork.

Build your own AI SRE agents. The open source toolkit for the AI era ✨

OpenSRE is an open-source framework for building AI Site Reliability Engineering agents that can investigate and resolve production incidents on your own infrastructure. It ships with integrations for 60+ observability and incident tools (Datadog, Grafana, Slack, PagerDuty-style alerting, etc.), a workflow definition layer, and — crucially — a training and evaluation environment designed for reinforcement-learning-style improvement of SRE agents.

The project frames itself as the 'missing SWE-bench for production incidents': while coding agents have benefited from scalable benchmarks and feedback, production debugging across logs, metrics, traces and runbooks has lacked an equivalent open substrate. OpenSRE provides that substrate, combining tool connectors, agent scaffolding, and an evaluation harness so teams can define their own workflows and measure agent performance over time.

It is trending with SRE and platform-engineering audiences who want an auditable, self-hosted alternative to closed AIOps products, and with researchers who need a realistic environment for agentic incident-response experiments. The project is in public alpha, so expect fast iteration on APIs and integrations.

DFlash: Block Diffusion for Flash Speculative Decoding

DFlash is a research project from Z-Lab that applies block diffusion models to speculative decoding, aiming to speed up large language model inference without sacrificing quality. Instead of relying on a small autoregressive draft model, DFlash drafts multiple tokens in parallel using a lightweight diffusion-based drafter, then verifies them against the target model — giving high acceptance rates and strong throughput gains.

The repo ships pretrained DFlash drafters for a wide range of popular open models, including Qwen3 and Qwen3.5 variants (4B through 35B-A3B), Qwen3-Coder, Kimi-K2.5, Llama-3.1-8B-Instruct, and both sizes of gpt-oss (20B and 120B). A training recipe is promised so users can produce custom drafters for their own target LLMs, and an accompanying paper and blog post describe the block-diffusion formulation in detail.

It is trending because speculative decoding is one of the highest-leverage inference optimizations for modern LLM serving, and DFlash plugs into an unusually broad list of open models out of the box — making it immediately useful for inference-infrastructure teams and LLM researchers chasing faster tokens-per-second.

T3 Code, from Theo Browne's ping.gg team, is a minimal web/desktop GUI wrapper around coding agents — currently OpenAI's Codex CLI and Anthropic's Claude Code, with more providers planned. It gives developers a lightweight graphical front-end over the authenticated CLI tools they already have installed, so agent sessions can be driven from a polished UI instead of a terminal.

Distribution is deliberately frictionless: you can run it instantly with npx t3, install the desktop build from GitHub Releases, or grab it from winget, Homebrew, or the AUR. The repo is explicitly early-stage — contributions are not yet being accepted — but it already includes an observability guide and clear provider setup instructions for Codex and Claude.

It is trending on the strength of the ping.gg / T3 brand and because it scratches a real itch: many developers want a nicer front door to Codex and Claude Code without committing to a heavyweight IDE plugin, and T3 Code positions itself as exactly that minimalist GUI layer.

Chrome DevTools for coding agents

chrome-devtools-mcp is an official project from the Chrome DevTools team that exposes a live Chrome browser to coding agents — such as Gemini, Claude, Cursor, or GitHub Copilot — through a Model Context Protocol (MCP) server. It effectively gives any MCP-compatible AI assistant the full power of Chrome DevTools: recording performance traces, inspecting network requests, capturing screenshots, reading console messages with source-mapped stack traces, and driving the browser via Puppeteer.

Beyond MCP, the repository also provides a standalone CLI for scripted use without an agent, plus detailed design-principles and troubleshooting documentation. Because automation runs against a real Chrome instance, agents can perform reliable end-to-end debugging and performance analysis on actual web apps — closing a long-standing gap where coding agents could reason about source code but not observe runtime behavior in a browser.

It is trending because MCP adoption is accelerating across agent frontends and because this is a first-party offering from the DevTools team itself, making it the canonical way to connect AI coding assistants to Chrome for front-end debugging, performance triage, and UI automation workflows.
